# AICC 验收原则

## 目标

本文档的目标是定义 AICC 验收测试的设计原则、整体目标、分层优先级、Mock 阶段确定性要求和真实模型调用边界。实现完成后，后续每组用例都能明确自己属于 L1/L2/L3/L4 哪一层、是否允许访问真实模型、是否阻塞合入或发布。

## 设计原则

1. L1/L2/L3 默认使用 Mock 模型，必须确定性执行，不访问真实模型，不依赖真实 API key。
2. L4 使用真实模型，只断言协议事实、任务状态、artifact、usage、trace 和错误分类，不断言自然语言全文。
3. 真实模型调用必须显式配置 `allow_real_model_calls=true`，缺 key 的普通开发验收用例标记为 `skipped`。
4. P0 覆盖核心协议、路由、任务、安全、usage 和 Mock Provider 行为，L1/L2/L3 P0 必须稳定通过。
5. P1 覆盖完整功能面，可阶段性 pending，但报告必须标注缺口。
6. P2 覆盖真实模型、性能、兼容性和发布验收，不阻塞普通开发合入。
7. 所有报告、trace、Provider 请求摘要和日志摘要不得泄露 key、token、原始敏感资源内容。

## 对应用例执行方式

本文档定义验收原则，不直接对应可执行用例。实现或评审相关用例时，应以本文档作为约束来源。
